# DroneOps release checklist

1. Create a release branch from the current working version.
2. Apply one focused product change per release.
3. Verify the preview build before merging.
4. Open the mobile route and test the changed workflow.
5. Merge only a checked version into the live branch.
6. Confirm the live deployment is ready.
7. Open the live route again and record the delivered version.

A delivery note must include the version, change summary, tested link, test results, known limitations and next improvement.
